# ReserveFlow Design Direction

ReserveFlow should look like a focused scheduling product, not a clone of the earlier portfolio projects.

## Principles

- Calm first impression, with enough color to feel productized.
- Clear booking CTA and low-friction service discovery.
- Operational dashboard density without visual heaviness.
- Calendar and time-slot motifs used as interface structure, not decoration.
- Classic typography and restrained motion for a recruiter-facing project.

## Palette

- Primary: deep teal blue for navigation, headings, and primary actions.
- Secondary: clear sky blue for panels, links, and availability states.
- Accent: warm amber for selected time slots and booking momentum.
- Neutral: warm white backgrounds with slate text for readability.

## Interaction notes

- Public pages should lead users toward `/booking`.
- Dashboard screens should make status changes obvious.
- Cards should stay compact with stable dimensions.
- Avoid a dark UI; this project should be brighter than API Meter and more operational than Luma Studio.

## Recruiter signal

The design should make it obvious that the boilerplate can support a real appointment product: public conversion flow, authenticated operations, database-backed entities, emails, and production deployment.
